Monte Carlo Simulation for Finance¶

In this assignment, we'll work through an example using Monte Carlo simulation for finance. We were introduced to the idea in a video on this module. In this notebook, we will work through a code example and see it in action!

Learning objectives:

  • Understand when and why Monte Carlo simulation is used
  • Run a toy simulation to estimate Pi
  • Understand the benefits of simulation vs. a single calculation
  • Be able to apply simulation to an investment problem

Recall that simulation is the approximation of the operation of a phenomenon of interest. That is, we use simulation to better understand a dynamic process by approximating how it might unfold. Monte Carlo simulation is a particular kind of simulation where we repeat a simulation many times, through random sampling, in order to understand a process that would otherwise be hard to solve.

We can use Monte Carlo simulation to understand the likelihood of certain outcomes from our process. In finance, that process might be something like how our portfolio will behave over time or to how to price options.

1) Estimating Pi using Monte Carlo methods¶

Before we apply Monte Carlo simulation to finance, let's get a feel for how Monte Carlo methods work. We'll be using them to estimate Pi ($\pi$), that reliable constant from geometry. Recall that $\pi$ is used in equations that relate the area and circumference of a circle. For example, the area of a circel is: $A_{circle} = \pi r^2$

A classical example for demonstrating Monte Carlo methods is to imagine we're trying to estimate the numerical value of $\pi$. Assume we know the equation, but don't know the actual numerical value of $\pi$.

To estimate $\pi$, we might start with a square board and throw imaginary darts at it. We'll want to begin by throwing our darts in a uniformly random way. Let's set up the code.

How will these uniformly random points inside of a square help us? We can use a bit of geometry. Imagine that we put a circle inside of the square. The diameter of the circle will be as wide as the square. We know the area of a square, $A_{square} = 2r^2$, where $r$ is the radius of the circle.

The area of the square is 1, because each side is of length 1 unit. The area of the inscribed circle (the circle inside the square) is then $\pi r^2 = \pi (\frac{1}{2})^2$.

Remember that $r = \frac{1}{2}$ because that's the radius of the circle, or one half the length of the square.

We can then set up this ratio: $$\frac{\text{area of circle}}{\text{area of square}} = \frac{\text{number of randomly generated points inside circle}}{\text{number of randomly generated points inside square}} $$

Then,

$$ \frac{\text{area of circle}}{\text{area of square}} = \frac{\pi r^2}{(2r)^2} = \frac{\pi r^2}{4(r^2)}$$

We can solve for $\pi$:

$$ \pi = 4* \frac{\text{area of circle}}{\text{area of square}}$$

To estimate $\pi$, we can look at our randomly generated points and check whether they're inside of the circle or not. Then we can divide the number of points inside the circle by all points, and multiply by 4 to estimate $\pi$. Note that we're estimating $\pi$, not calculating the exact value. That is the heart of simulation, which uses algorithms such as this one to estimate values that might be too hard to calculate exactly otherwise.

We can check whether a point is inside of the circle by checking whether $x^2 + y^2 \leq 1$. We can then count up the points inside the circle and divide by the total points.

2) Monte Carlo methods in finance¶

We will now use the Monte Carlo simulation method from above to simulate a portfolio return. We need to know a few things:

  • The annual return rate of our portfolio
  • The volatility of the return, measured in the standard deviation of the portfolio
  • How much investment we're adding each year

Assume our strategy is to use to a systematic investment plan, where we add a fixed amount of new investment every year. This way, we can grow our portfolio slowly over time and take advantage of compounding and dollar-cost averaging to reduce the negative impact volatility can have on our investment over a long time horizon.

For our example, let's take a return rate of 7% and a standard deviation of 11%, with a starting investment of 10,000 dollars and a yearly additional investment of another 10,000 dollars.

2a) Run the simulation one time¶

Let's now run through our simulation one time to see what happens to our investment after 10 years. This is not a Monte Carlo simulation yet as we're only running through it once! Recall that a Monte Carlo simulation is better than a single simulation because it involves running through our model multiple times, refreshing our random variables each time, and looking at a summary statistic. We'll get to the Monte Carlo version in the next step, so hold on!

def calculate_returns(starting_investment, yearly_investment, expected_r, volatility, years):
    total_investment = starting_investment 
    
    for year in range(years):
        market_return = np.random.normal(expected_r, volatility)
        ending_value = yearly_investment + (1+ market_return)* total_investment
        print("Year ", year+1, ": return = ", market_return, " and year-end value = ", f"{ending_value:,.0f}")
        total_investment = ending_value

# run this cell for the helper function 

def run_monte_carlo(n, starting_investment, yearly_investment, expected_r, volatility, years): 
    returns_list = []
    
    for _ in range(n):
        total_investment = starting_investment
        returns = []
        
        for year in range(years):
            market_return = np.random.normal(expected_r, volatility)
            ending_value = yearly_investment + (1 + market_return) * total_investment
            total_investment = ending_value
            returns.append(total_investment)
            
        returns_list.append(returns)
    
    returns_df = pd.DataFrame(returns_list, columns=["y1","y2","y3","y4","y5","y6","y7","y8","y9","y10"])
    return returns_df

2c) Calculate summary statistics of Monte Carlo simulation¶

In the above step, we ran our Monte Carlo simulation 500 times. The point of such simulation is to then take a look at the summary statistics. These summary statitics -- for example, the mean of returns after year 10 -- will be more accurate than a single run of the simulation, as we did in step 2a.

The mean of 500 runs is better than one run because each run has a random component. A mean of many random runs is more accurate because a single run could have been really high or really low, just by chance.
